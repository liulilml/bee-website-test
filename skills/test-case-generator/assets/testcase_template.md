## 测试点设计说明

### 命名规范
```
TP-[模块缩写]-[序号]: [操作对象] + [操作动作] + [预期结果/验证点]
```

示例：
- `TP-VIP-001: 未订阅用户点击订阅按钮，正确跳转支付页`
- `TP-VIP-002: 已订阅用户查看权益页，显示正确的到期时间`
- `TP-VIP-003: 会员过期后访问专属功能，提示续费`

### 优先级定义
| 级别 | 含义 | 适用场景 |
|------|------|----------|
| P1 | 最高 | 核心主流程，阻塞性验证 |
| P2 | 高 | 重要功能、常见异常 |
| P3 | 中 | 边界条件、次要流程 |
| P4 | 低 | 极端场景、兼容性 |

### 测试方法
- **等价类划分**：有效等价类（符合规则）+ 无效等价类（违反规则）
- **边界值分析**：最小值、最小值 -1、最小值 +1、最大值、最大值 -1、最大值 +1、空值、零值
- **状态转换**：每个合法状态转换路径 + 非法状态转换尝试
- **场景法**：主流程（Happy Path）+ 分支流程 + 异常中断与恢复
