# 设计稿图片解析 Prompt 指引

## 用途

当用户提供 UI 设计稿、原型图、界面截图时，使用此 prompt 指导 AI 分析图片并提取测试相关信息。

## 核心 Prompt

```
请分析这张 UI 设计稿/原型图，提取以下信息用于生成测试点：

## 1. 页面元素识别
- 所有可见的 UI 控件（按钮、输入框、下拉框、复选框、单选框、开关、滑块等）
- 文本内容（标题、标签、提示文案、错误提示等）
- 列表/表格（列名、排序、分页等）
- 图片/图标（功能图标、状态图标、装饰图标等）
- 导航元素（菜单、面包屑、Tab 切换等）

## 2. 交互流程分析
- 用户可执行的操作（点击、输入、选择、拖拽、滑动等）
- 操作后的反馈（弹窗、跳转、动画、状态变化等）
- 页面跳转关系（从哪来到哪去）
- 表单提交流程（填写→验证→提交→结果）

## 3. 业务规则推断
- 输入验证规则（必填、格式、长度、唯一性等）
- 状态展示规则（不同状态下的 UI 表现）
- 权限控制（不同角色看到的内容差异）
- 数据展示规则（排序、过滤、分组等）

## 4. 边界条件识别
- 数值范围（最大值、最小值、步长）
- 数量限制（最多选择 X 项、最多上传 Y 个文件）
- 时间限制（有效期、截止时间、超时处理）
- 字符限制（最大长度、允许的字符集）

## 5. 异常场景推断
- 网络异常（加载失败、提交失败、重试机制）
- 数据异常（空数据、脏数据、重复数据）
- 操作异常（重复点击、快速切换、中断操作）
- 权限异常（未登录、权限不足、登录过期）

## 6. 兼容性考虑
- 不同屏幕尺寸的适配
- 深色模式/浅色模式
- 不同浏览器的表现差异

## 输出格式

请按以下格式输出分析结果：

### 页面名称
[页面/模块名称]

### 功能列表
1. [功能 1 名称]
   - 元素：[元素列表]
   - 操作：[可执行的操作]
   - 规则：[业务规则]

2. [功能 2 名称]
   ...

### 边界条件
- [边界条件 1]
- [边界条件 2]
...

### 异常场景
- [异常场景 1]
- [异常场景 2]
...

### 测试点建议
基于以上分析，建议覆盖以下测试点：
- 正向：[X 个测试点]
- 反向：[X 个测试点]
- 边界：[X 个测试点]
- 特殊：[X 个测试点]
```

## 使用示例

### 用户输入
```
帮我拆测试点，这是会员订阅页面的设计稿
[上传图片]
```

### AI 执行
1. 调用 `image` 工具，使用上述 prompt 分析图片
2. 提取功能列表、交互流程、业务规则、边界条件、异常场景
3. 基于分析结果生成结构化测试点
4. 创建飞书文档并发送链接给用户确认

## 注意事项

1. **图片质量**：如果图片模糊或分辨率低，先告知用户可能影响识别准确性
2. **多页面**：如果用户提供了多张设计稿，逐张分析后汇总
3. **标注批注**：如果设计稿包含标注/批注，优先提取标注信息（更准确）
4. **状态展示**：如果设计稿展示了多个状态（如空状态、加载状态、错误状态），分别分析
5. **交互说明**：如果设计稿包含交互说明文档，结合文档分析更准确

## 交叉验证

图片分析结果应与用户提供的文字需求交叉验证：
- 图片中有但需求没提到的 → 询问用户是否需要覆盖
- 需求有但图片没展示的 → 询问用户是否有对应设计稿
- 两者不一致的 → 向用户确认以哪个为准
